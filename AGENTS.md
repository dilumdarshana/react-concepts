# react-concepts

Learning monorepo with 5 independent React 19 projects. Each is fully self-contained in its own subdirectory.

## Project layout

| Directory | Stack |
|-----------|-------|
| `basics/` | hooks, context, react-router v7, react-hook-form + zod |
| `react-zustand/` | Zustand state management |
| `RTK/` | Redux Toolkit + shadcn/ui + Tailwind v4 + lucide-react |
| `SSR/` | Express-based SSR (streaming + non-streaming) |
| `tanstack/` | TanStack Query + Table + Router (file-based) |

## Commands

Run all commands **inside** the relevant subdirectory (`cd basics/ && ...`).

| Command | Action |
|---------|--------|
| `pnpm dev` | Start dev server |
| `pnpm build` | `tsc -b && vite build` (except SSR, see below) |
| `pnpm lint` | `eslint .` (flat config, `eslint.config.js`) |
| `pnpm preview` | `vite preview` |

### SSR differences

- `pnpm dev` runs `node server_stream --watch` (Node.js server, not Vite's dev server)
- `pnpm build` = `pnpm build:client` (`vite build --outDir dist/client`) + `pnpm build:server` (`vite build --ssr src/entry-server.tsx --outDir dist/server`)
- `pnpm preview` = `cross-env NODE_ENV=production node server`
- Switch between streaming / non-streaming by editing `package.json` dev script to point at `server_stream.js` or `server.js`

## Generated code

`tanstack/src/routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin/vite`. **Never edit it manually.**

## Environment

- Node: v24.15.0 (`.nvmrc`)
- Package manager: pnpm (no npm/yarn)
- No tests, no CI/CD

## Style notes

- ESLint flat config (`eslint.config.js`) in all projects
- Tailwind CSS v4 via `@tailwindcss/vite` plugin (no `postcss.config.js` / `tailwind.config.js`). Used in: `RTK/`, `tanstack/`
- shadcn/ui (New York style, `@/` path alias → `src/`) in `RTK/` only
- SSR uses `hydrateRoot` on client, `renderToString` or `renderToPipeableStream` on server
